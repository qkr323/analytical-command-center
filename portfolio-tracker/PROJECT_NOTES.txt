WHAT IS THIS?
-------------
A personal portfolio management web app that aggregates all your brokerage accounts
into one dashboard with compliance monitoring, live prices, and allocation charts.

Live dashboard: https://analytical-command-center.vercel.app


BROKERS CONNECTED
-----------------
  IBKR       — Flex Query API (automatic, no local app needed)
  Futu       — Daily PDF statement via Gmail (automatic via sync_from_email.py)
               Fallback: Futu OpenD local daemon (see HISTORY section below)
  Binance    — REST API via local backend (geoblocked on Railway — must run from Mac)
  SoFi       — Daily/monthly PDF statement via Gmail (automatic via sync_from_email.py)
  OSL        — PDF upload only (manual, monthly)
  Hang Seng  — PDF upload only (manual, monthly — download from Hang Seng online banking)


TECH STACK
----------

  Auth:      API key (X-API-Key header) — injected server-side, never exposed to browser

GMAIL-BASED SYNC (Futu + SoFi)
--------------------------------
  Both Futu HK and SoFi HK send daily statement emails with PDF attachments to
  sam.sanghyunpark@gmail.com. A Gmail filter auto-forwards these to qkr323@gmail.com.

  The script backend/sync_from_email.py:
    - Connects to Gmail API (sam.sanghyunpark@gmail.com) via OAuth2
    - Searches for unprocessed Futu and SoFi statement emails
    - Downloads the PDF attachment
    - Uploads it to /upload/statement on the local backend
    - Labels the email "portfolio-synced" to avoid re-processing

  Gmail credentials:

KNOWN QUIRKS
------------
  - HK stock prices sometimes fail in yfinance (ticker format issues) — prices show last known value
  - Futu daily PDF statements have long ETF names that wrap across lines — the parser handles this
    by detecting unclosed parentheses and joining with the next line before applying regex
  - Python 3.14 is too new — always use the .venv (Python 3.12) for the local backend
  - Hang Seng statement date format is DDMMMYYYY (e.g. 27MAR2026) — parsed by HangSengParser
  - Hang Seng ETFs 2800 (Tracker Fund) and 3416 (HSCEI ETF) are approved as broad index ETFs
  - protobuf conflict: google-api-python-client requires protobuf 7.x but futu-api requires 3.x.
    Both are installed in .venv — sync_from_email.py works fine; Futu OpenD sync may break if used.
  - Binance is geoblocked on Railway (HTTP 451) — must sync from Mac terminal via local backend


HISTORY — WHAT WE TRIED AND WHY WE CHANGED
--------------------------------------------
  FUTU SYNC — v1: Futu OpenD local daemon
    Originally, Futu positions were synced via the Futu OpenD application — a local program that
    runs on the Mac at 127.0.0.1:11111 and exposes a Python API (futu-api package). The backend
    connected to it via routers/sync.py → services/brokers/futu_opend.py.

    Problem: OpenD only runs locally on the Mac. Railway's cloud backend at 127.0.0.1 refers to
    Railway's own container, not the user's Mac — so the website sync button always failed.
    Fix applied: the sync button showed a friendly error directing to SYNC_INSTRUCTIONS.txt.
    The workflow was: start local backend → curl /sync/futu → OpenD pulls data → writes to Railway DB.

    This worked but was cumbersome — required OpenD running, local backend running, and terminal commands.

  FUTU SYNC — v2: Gmail PDF statement (current)
    Futu HK sends a daily PDF statement to sam.sanghyunpark@gmail.com each morning. A Gmail filter
    forwards these to qkr323@gmail.com. The sync_from_email.py script reads the email, downloads the
    PDF, and uploads it to /upload/statement.

    The Futu PDF parser (services/parsers/futu.py) was rewritten to handle the daily statement format:
    - Uses text-based parsing (pdfplumber extracts each row as a separate table with no headers)
    - Two-step regex: _POS_SYM matches the symbol, _POS_DATA matches exchange/currency/price/value
    - _join_wrapped_lines() handles long ETF names that wrap across lines (e.g. TLT, EDV, KWEB, LIT)
    - Prefers "Ending Assets Overview" (end-of-day prices) over "Starting Assets Overview"

  BINANCE SYNC — geoblocked on Railway
    Binance.com is blocked on Railway's US/EU infrastructure (HTTP 451 Unavailable For Legal Reasons).
    The website sync button was returning 500 errors. Fixed with a friendly error message in SyncPanel.tsx.
    Binance must be synced from Mac terminal via local backend.
